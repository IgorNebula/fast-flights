## Описание

### Цель задачи

Создать мобильное демо-приложение для получения и отображения стоимости перелетов на основе фиктивных данных, сгенерированных программно и хранящихся в оперативной памяти.

### Требования

1. Приложение должно отображать интерфейс пользователя согласно прототипу в docs\analisys_design\interface_prototype
2. Данные о перелетах должны генерироваться программно при запуске приложения
3. Данные должны храниться только в оперативной памяти (не использовать постоянное хранилище)
4. Приложение должно поддерживать основные функции просмотра информации о перелетах
5. Интерфейс должен быть адаптирован для мобильных устройств

### Ограничения

1. Не использовать реальные данные о перелетах
2. Не подключаться к внешним API для получения данных
3. Все данные должны быть фиктивными и генерироваться программно
